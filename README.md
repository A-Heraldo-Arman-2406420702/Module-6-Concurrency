## Commit 1 Reflection notes
Fungsi `handle_connection` bekerja dengan menerima `TcpStream` yang mewakili koneksi terbuka antara klien dan server. Kita menggunakan `BufReader` untuk membungkus `stream` tersebut guna meningkatkan efisiensi pembacaan data, karena ia melakukan *buffering* alih-alih membaca byte demi byte secara langsung dari OS. Metode `.lines()` memecah aliran data setiap kali menemukan karakter *newline*, yang sangat ideal untuk mem-parsing protokol HTTP di mana setiap *header* dipisahkan oleh baris baru. Iterator kemudian di-map untuk menangani potensi *error* (via `unwrap`) dan kita menggunakan `take_while` untuk menghentikan pembacaan saat menemukan baris kosong (`!line.is_empty()`), yang dalam spesifikasi HTTP menandakan akhir dari bagian *header* sebelum *body* dimulai. Data tersebut kemudian dikumpulkan ke dalam vektor `http_request` untuk dicetak, memungkinkan kita melakukan inspeksi langsung terhadap *raw request* dari *browser*.

## Commit 2 Reflection notes
![gambar](/assets/images/commit-2.png)

Pada tahapan ini, `handle_connection` dimodifikasi untuk tidak hanya membaca *request*, tetapi juga mengirimkan respons HTTP yang valid kembali ke klien. Kode membaca isi file `hello.html` menggunakan `fs::read_to_string` dan menghitung panjang *byte*-nya (`length`) sebagai *metadata* yang krusial. Respons HTTP dibentuk menggunakan makro `format!` yang merangkai `status_line` (menandakan kode sukses 200), *header* `Content-Length`, dan `contents` (isi file HTML itu sendiri). Pemisah `\r\n\r\n` (CRLF ganda) digunakan secara ketat sesuai dengan protokol HTTP untuk memisahkan bagian *header* dan *body* respons. Akhirnya, string respons tersebut dikonversi menjadi *byte array* menggunakan `as_bytes()` dan ditulis kembali ke `TcpStream` via `write_all`, yang kemudian di-*render* oleh *browser* pengguna.
